The Certiva API is served by the NestJS application in `apps/api`. All routes are prefixed with `/api`.

Authentication uses an `httpOnly` cookie named `certiva_access_token` set on successful login.

## Authentication

### Login

```
POST /api/auth/login
```

**Body:**

```json
{
  "username": "string",
  "password": "string"
}
```

Sets `certiva_access_token` cookie on success. Rate limited: 5 requests per 60 seconds by default.

### Get current admin

```
GET /api/auth/me
```

Returns the authenticated admin's profile.

### Register admin

```
POST /api/auth/register
```

Requires `OWNER` or `SUPER_ADMIN` role.

### List admins

```
GET /api/auth/admins
```

Requires `OWNER` or `SUPER_ADMIN` role.

### Update admin

```
PATCH /api/auth/admins/:id
```

Requires `OWNER` or `SUPER_ADMIN` role.

### Delete admin

```
DELETE /api/auth/admins/:id
```

Requires `OWNER` or `SUPER_ADMIN` role. Fails if the admin has historical activity — disable instead.

## Credentials

All credential endpoints require authentication.

| Method | Path | Description |
|---|---|---|
| `POST` | `/api/credentials` | Issue a single credential |
| `POST` | `/api/credentials/bulk` | Issue credentials from a CSV file (multipart) |
| `POST` | `/api/credentials/bulk-revoke` | Revoke multiple credentials |
| `POST` | `/api/credentials/bulk-delete` | Delete multiple credentials |
| `GET` | `/api/credentials` | List credentials |
| `GET` | `/api/credentials/:id` | Get credential by ID |
| `DELETE` | `/api/credentials/:id` | Delete credential |
| `PATCH` | `/api/credentials/:id/revoke` | Revoke a credential with reason |
| `POST` | `/api/credentials/:id/rebuild-assets` | Rebuild QR code and metadata |
| `POST` | `/api/credentials/:id/secure-pdf` | Attach a secure PDF to credential |
| `GET` | `/api/credentials/:id/metadata` | Retrieve credential metadata |
| `GET` | `/api/credentials/:id/qr` | Retrieve QR code image |

## Verification (public)

These endpoints do not require authentication. Rate limits apply.

| Method | Path | Rate limit | Description |
|---|---|---|---|
| `POST` | `/api/verify/credential/code` | 30 req / 60 s | Verify by verification code |
| `POST` | `/api/verify/credential/pdf` | 10 req / 60 s | Verify credential by PDF upload |
| `POST` | `/api/verify/secure-pdf` | 10 req / 60 s | Verify a secure PDF |
| `GET` | `/api/verify/:verificationId` | — | Get verification result by ID |
| `GET` | `/api/verify/:verificationId/certificate` | — | Download certificate PDF |
| `POST` | `/api/verifications` | 30 req / 60 s | General verification endpoint |
| `POST` | `/api/verification/upload` | 10 req / 60 s | Upload-based verification |

## Verification result values

| Value | Meaning |
|---|---|
| `VALID` | Credential is active and unrevoked |
| `REVOKED` | Credential has been revoked |
| `NOT_FOUND` | No matching credential found in the registry |
| `INVALID` | Document hash does not match the stored proof |
