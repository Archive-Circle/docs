Certiva validates every environment variable at startup using a Zod schema. Missing or invalid values cause a hard failure with a descriptive error message.

## Root-level variables

Set in the root `.env` file (Docker Compose and Turborepo shared context).

| Variable | Required | Default | Description |
|---|---|---|---|
| `POSTGRES_USER` | Yes | — | PostgreSQL username |
| `POSTGRES_PASSWORD` | Yes | — | PostgreSQL password |
| `POSTGRES_DB` | Yes | — | PostgreSQL database name |
| `NODE_ENV` | Yes | `development` | Framework mode (`development` / `production` / `test`) |
| `APP_ENV` | Yes | `development` | Deployment context (`development` / `staging` / `production`) |

## API variables

Set in `apps/api/.env`.

### Database and cache

| Variable | Required | Example | Description |
|---|---|---|---|
| `DATABASE_URL` | Yes | `postgresql://user:pass@localhost:5432/certiva` | PostgreSQL 16 connection string |
| `REDIS_URL` | Yes | `redis://localhost:6379` | Redis 7 connection string |

### Authentication

| Variable | Required | Notes | Description |
|---|---|---|---|
| `JWT_SECRET` | Yes | Min 64 chars | Token signing secret. Placeholder values are rejected in staging/prod. |
| `JWT_EXPIRES_IN` | No | `12h` | Token lifetime |

### Server

| Variable | Required | Default | Description |
|---|---|---|---|
| `PORT` | No | `4000` | API listening port |
| `CORS_ORIGINS` | Yes | — | Comma-separated origin allowlist. Wildcard rejected in staging/prod. |
| `COOKIE_SECURE` | No | `false` | Set `true` when serving over HTTPS |
| `TRUST_PROXY` | No | `false` | Set `true` when behind a reverse proxy or Cloudflare |

### Public URLs

| Variable | Required | Description |
|---|---|---|
| `API_PUBLIC_BASE_URL` | Yes | Used in QR codes and verification link generation |
| `WEB_PUBLIC_BASE_URL` | Yes | Used in credential metadata URIs |

### Storage

| Variable | Required | Options | Description |
|---|---|---|---|
| `STORAGE_DRIVER` | Yes | `local` / `r2` | Asset storage driver. `r2` is enforced in staging/prod. |
| `ASSET_STORAGE_ROOT` | No | — | Local filesystem root path (local driver only) |

### Cloudflare R2

Required when `STORAGE_DRIVER=r2`.

| Variable | Description |
|---|---|
| `R2_ACCOUNT_ID` | Cloudflare account ID |
| `R2_BUCKET` | R2 bucket name |
| `R2_ACCESS_KEY_ID` | R2 access key |
| `R2_SECRET_ACCESS_KEY` | R2 secret key |
| `R2_ENDPOINT` | R2 S3-compatible endpoint URL |
| `R2_PUBLIC_BASE_URL` | Public CDN base URL for assets |
| `R2_FORCE_PATH_STYLE` | `true` / `false` (path-style vs. virtual-hosted requests) |

### Blockchain (optional)

| Variable | Required | Description |
|---|---|---|
| `BLOCKCHAIN_ENABLED` | No | `true` to enable anchoring. Requires the variables below. |
| `POLYGON_AMOY_RPC_URL` | If enabled | RPC endpoint for Polygon Amoy testnet |
| `PRIVATE_KEY` | If enabled | `0x`-prefixed 64-char hex signing key. Dev keys rejected in staging/prod. |
| `CONTRACT_ADDRESS` | If enabled | `0x`-prefixed 40-char deployed contract address |
| `ISSUER_WALLET` | If enabled | `0x`-prefixed 40-char issuer wallet address |
| `BLOCKCHAIN_RPC_TIMEOUT_MS` | No | RPC call timeout in milliseconds (default: `12000`) |

### Rate limiting

All rate-limit variables are optional. Redis-backed rate limiting is recommended in staging/production.

| Variable | Default | Description |
|---|---|---|
| `RATE_LIMIT_STORE` | `memory` | `redis` or `memory` |
| `RATE_LIMIT_REDIS_PREFIX` | — | Redis key namespace prefix |
| `RATE_LIMIT_AUTH_LOGIN_MAX` | `5` | Max login attempts per window |
| `RATE_LIMIT_AUTH_LOGIN_WINDOW_SECONDS` | `60` | Login rate-limit window |
| `RATE_LIMIT_VERIFY_MAX` | `30` | Max verification requests per window |
| `RATE_LIMIT_VERIFY_WINDOW_SECONDS` | `60` | Verification rate-limit window |
| `RATE_LIMIT_VERIFY_UPLOAD_MAX` | `10` | Max file-upload verification requests per window |
| `RATE_LIMIT_VERIFY_UPLOAD_WINDOW_SECONDS` | `60` | Upload rate-limit window |
| `RATE_LIMIT_ADMIN_MAX` | `100` | Max admin route requests per window |
| `RATE_LIMIT_ADMIN_WINDOW_SECONDS` | `60` | Admin rate-limit window |

## Web variables

Set in `apps/web/.env`.

| Variable | Exposed to browser | Description |
|---|---|---|
| `NODE_ENV` | No | Build mode |
| `APP_ENV` | No | Deployment context |
| `INTERNAL_API_URL` | No | Server-side API URL used by BFF routes |
| `COOKIE_SECURE` | No | `true` for HTTPS |
| `NEXT_PUBLIC_SITE_URL` | Yes | Public-facing site URL |
| `NEXT_PUBLIC_SUPPORT_EMAIL` | Yes | Support contact email shown in the UI |

<Warning>
Variables prefixed with `NEXT_PUBLIC_` are embedded into the browser bundle at build time. Never store secrets, API keys, or internal URLs in `NEXT_PUBLIC_` variables.
</Warning>

## Worker variables

Set in `apps/worker/.env`.

| Variable | Required | Description |
|---|---|---|
| `DATABASE_URL` | Yes | PostgreSQL connection string |
| `REDIS_URL` | Yes | Redis connection string |
| `BLOCKCHAIN_ENABLED` | No | `true` to enable blockchain anchoring jobs |
| `POLYGON_AMOY_RPC_URL` | If enabled | RPC endpoint |
| `PRIVATE_KEY` | If enabled | Signing key |
| `CONTRACT_ADDRESS` | If enabled | Contract address |
| `ISSUER_WALLET` | If enabled | Issuer wallet address |
| `BLOCKCHAIN_RPC_TIMEOUT_MS` | No | RPC timeout in milliseconds |

## Validation rules

Certiva enforces these constraints at startup (applied in staging and production):

- `JWT_SECRET` must be at least 64 characters and must not be a known placeholder value.
- `CORS_ORIGINS` must not be empty or contain a wildcard (`*`).
- `STORAGE_DRIVER` must be `r2` (local storage is rejected).
- `PRIVATE_KEY` must not match any known development test keys.
- When `BLOCKCHAIN_ENABLED=true`, `POLYGON_AMOY_RPC_URL`, `PRIVATE_KEY`, and `CONTRACT_ADDRESS` are all required.
- When `STORAGE_DRIVER=r2`, all `R2_*` variables are required.
