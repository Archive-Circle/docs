<Info>
  **Prerequisites**:
  - Node.js 20 or newer
  - npm
  - A local clone of this repository
  - A `docs.json` file at the repository root
</Info>

This page explains how to preview and validate the Archive Circle Mintlify docs site locally.

## Install the Mintlify CLI

Install the CLI globally:

```bash
npm install -g mintlify
```

Check the installed version:

```bash
mintlify --version
```

<Note>
Some Mintlify documentation refers to the newer `mint` command. This repository currently uses the `mintlify` command.
</Note>

## Preview locally

From the repository root, run:

```bash
mintlify dev --no-open
```

Or open the browser automatically:

```bash
mintlify dev
```

The preview usually runs at:

```text
http://localhost:3000
```

If port `3000` is already in use, Mintlify may select another available port. Check the terminal output for the exact URL.

## Validate navigation references

Before opening or merging a PR, make sure every page listed in `docs.json` exists.

Run this from the repository root:

```bash
python3 - <<'PY'
import json
from pathlib import Path

cfg = json.load(open('docs.json'))
missing = []
for tab in cfg.get('navigation', {}).get('tabs', []):
    for group in tab.get('groups', []):
        for page in group.get('pages', []):
            if isinstance(page, str) and not Path(page + '.mdx').exists():
                missing.append(page)

print('site name:', cfg.get('name'))
print('tabs:', [tab.get('tab') for tab in cfg.get('navigation', {}).get('tabs', [])])
print('missing nav refs:', missing or 'none')
raise SystemExit(1 if missing else 0)
PY
```

## Multi-product docs checks

Archive Circle docs are structured as a multi-product site.

Before merging structural docs changes, confirm:

- New products have their own product-level tab in `docs.json`.
- New products use product-prefixed folders, for example `chainlens/`.
- Product-specific app or GitHub links are clearly labeled.
- The global site name remains company-level unless the whole docs brand changes.
- No unrelated product docs are mixed into another product's section.

## Code formatting

Use an editor that understands MDX. For VS Code, useful extensions include:

## Troubleshooting

<AccordionGroup>
  <Accordion title="The local preview does not start">
    Make sure you are running the command from the repository root where `docs.json` exists. Then check that Node.js and npm are installed.
  </Accordion>

  <Accordion title="A page returns 404 in local preview">
    Check that the page exists as a `.mdx` file and that its path is listed correctly in `docs.json` without the `.mdx` extension.
  </Accordion>

  <Accordion title="Mintlify CLI behavior looks stale">
    Update the CLI:

    ```bash
    npm update -g mintlify
    ```

    Then run `mintlify --version` again.
  </Accordion>

  <Accordion title="MDX fails to render">
    Check for unclosed Mintlify components, invalid JSX props, or code examples that need to be inside fenced code blocks.
  </Accordion>
</AccordionGroup>

## Deployment

Mintlify deploys from the default branch when the GitHub integration is configured. For pull requests, prefer reviewing the rendered preview when available. If the deployment check is skipped, run the local preview and validation commands above.
